<ProjectSection title="The Mission" number="01">

UK automotive dealers live in a data mess. Service histories are scattered across 44 manufacturer databases. HPI checks, finance records, and DVLA data live in separate silos. Mileage readings get lost between trades. Compliance paperwork? Done by hand.

**AutoProv fixesthat.** One platform connects everything — pulls official service records, flags odometer inconsistencies, runs Experian and DVLA checks, generates compliant PDI reports, builds Distance Sale Packs, and even handles AI-powered dispute responses under UK consumer law.

</ProjectSection>

<ProjectSection title="Multi-Source API Engine" number="02">

I built a **resilient ingestion layer** that queries multiple automotive APIs simultaneously, normalises conflicting schemas, and merges fragmented data into a single coherent record. Think of it as a universal translator for the UK automotive data ecosystem.

The real challenge? **Speed.** Some of these endpoints are slow by design. I optimised the async fetching pipeline — parallelised upstream requests, introduced data-caching layers, and cut API response times drastically. What used to take tens of seconds now resolves in moments, even under heavy traffic.

</ProjectSection>

<ProjectSection title="PDF Generation at Scale" number="03">

This was the hardest technical problem in the project. AutoProv needs to compile **massive, deeply nested datasets** into publication-quality PDF reports — without hitting serverless timeouts or leaking memory.

I engineered a custom PDF generation subsystem that structures multi-tiered data arrays into clean, readable multi-page layouts. Handles dense relational trees that would choke standard rendering pipelines. The output? Professional-grade reports that dealers can send straight to customers and regulators.

</ProjectSection>

<ProjectSection title="Billing & Monetisation" number="04">

Deployed a **full Stripe billing architecture** with monthly and yearly tiers. But the killer feature? An admin dashboard that lets project managers change pricing, update SKUs, and tweak subscription models — **without a single redeploy.** All synced live with Stripe via robust webhook handlers that track lifecycles, handle failures, manage trials, and keep the database in strict lockstep with Stripe's state.

</ProjectSection>

<ProjectSection title="Viral Growth Engine" number="05">

Added a **referral and rewards programme** — both client and server-side tracking. When a referred user converts, reward hooks fire automatically, attributing discounts or credits to the referrer's account. No manual intervention, no support tickets. Self-serve growth.

</ProjectSection>

<ProjectSection title="Engineering Impact" number="06">

I inherited a prototype that needed structural intervention. **Rewrote the core** — modularised the codebase, decoupled tangled components, eliminated technical debt. The result is a clean, production-hardened platform that scaled from zero to commercial success for the company.

</ProjectSection>
