<ProjectSection title="The Mission" number="01">

Enterprise finance teams track capital expenditure through spreadsheets, emails, and fragile approval chains. Budget requests get lost. Asset lifecycles go unmonitored. Financial reports take days to compile, and by the time they are ready, the numbers are stale.

**Capex replaced all of that.** A single real-time dashboard that tracks budgets from request to approval, monitors asset lifecycles end to end, and computes complex financial summaries instantly. No spreadsheets involved.

</ProjectSection>

<ProjectSection title="Full Rewrite, Modern Stack" number="02">

The legacy app was JavaScript with Material-UI. Slow builds, bloated bundles, and a design system that fought every change. I led a **complete ground-up rewrite** into TypeScript, Vite, and Tailwind CSS with shadcn/ui.

Build times dropped. Bundle sizes shrank. The shadcn/ui components gave us a composable design system that actually fit the product. Every pixel matched the client's raw Photoshop mocks. No compromises, no shortcuts.

</ProjectSection>

<ProjectSection title="Multi-Step Calculation Engine" number="03">

Capex runs on a **client-side financial engine** that pulls together department budgets, asset costs, depreciation schedules, and regulatory thresholds. It computes tax-like financial summaries in real time. Users navigate deeply dependent multi-step forms with interactive fields, dynamic select matrices, and custom range sliders.

**State orchestration was the hardest part.** So many interdependent fields across multiple steps that a single change ripples through the entire form. I used React Context API to cache progress across step boundaries. No state fragmentation, no unnecessary re-renders. Just smooth navigation through complex financial workflows.

</ProjectSection>

<ProjectSection title="Role-Based Access Control" number="04">

Enterprise finance data is sensitive. Budget figures, depreciation schedules, pricing configurations. Not everyone gets to see everything.

I designed a **two-tier RBAC system** with Admin and User roles. Every view, every data point, every config variable is gated behind token-verified authorization scopes. Admins see the full picture. Users see only what they need. No confusion, no data leaks.

</ProjectSection>

<ProjectSection title="Engineering Impact" number="05">

As **Frontend Lead**, I set the technical direction. Delegated milestones, managed features, and enforced TypeScript strict mode across the entire codebase. The result is smaller bundle sizes, faster computations, and a codebase that does not rot.

The client got a dashboard that loads faster, computes instantly, and handles whatever the finance team throws at it.

</ProjectSection>
