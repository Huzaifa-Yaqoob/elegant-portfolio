<ProjectSection title="The Mission" number="01">

Enterprise finance teams track capital expenditure through a nightmare of spreadsheets, disconnected emails, and fragile approval chains. Budget requests get lost. Asset lifecycles go unmonitored. Financial reports take days to compile — and by the time they're ready, the numbers are already stale.

**Capex replaced all of that.** A single, real-time dashboard that tracks budgets from request to approval, monitors asset lifecycles end-to-end, and computes complex financial summaries instantly — without a single spreadsheet in sight.

</ProjectSection>

<ProjectSection title="Full Rewrite, Modern Stack" number="02">

The legacy app was written in JavaScript with Material-UI — slow builds, bloated bundles, and a design system that fought every change. I led a **complete ground-up rewrite** into TypeScript, Vite, and Tailwind CSS with shadcn/ui.

The difference was dramatic. Build times collapsed. Bundle sizes shrank. And the shadcn/ui components gave us a lightweight, composable design system that actually fit the product. Every pixel was matched to the client's raw Photoshop mocks — no compromises, no shortcuts.

</ProjectSection>

<ProjectSection title="Multi-Step Calculation Engine" number="03">

Capex runs on a **client-side financial valuation engine** that aggregates disparate variables — department budgets, asset costs, depreciation schedules, regulatory thresholds — and computes tax-like financial summaries in real time. Users navigate deeply dependent multi-step forms with interactive text fields, dynamic select matrices, and custom range sliders.

The hardest part? **State orchestration.** With so many interdependent fields across multiple steps, a single change could ripple through the entire form. I used React Context API to cache progress across step boundaries — no state fragmentation, no unnecessary re-renders, just seamless navigation through complex financial workflows.

</ProjectSection>

<ProjectSection title="Role-Based Access Control" number="04">

Enterprise finance data is sensitive. Budget figures, depreciation schedules, pricing configurations — not everyone gets to see everything.

I designed a **two-tier RBAC system** with Admin and User roles. Every view, every data point, every configuration variable is gated behind token-verified authorization scopes. Admins see the full picture. Users see only what they need. No data leaks, no confusion.

</ProjectSection>

<ProjectSection title="Engineering Impact" number="05">

As **Frontend Lead**, I set the technical direction — delegated milestones, managed features, and enforced TypeScript strict mode across the entire codebase. The result? Dramatically smaller bundle sizes, faster runtime computations, and a codebase that's actually maintainable.

The client got a dashboard that loads faster, computes instantly, and scales to whatever their finance team throws at it.

</ProjectSection>
