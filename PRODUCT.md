## Brand Identity

**Brand Name:** Obsidian Edge
**Tagline:** "Built fast. Runs premium. Scales infinitely."
**Logo:** `H.` monogram (favicon) — a sharp Oxanium letterform on deep obsidian surface, embodying brutalist minimalism and zero-radius precision. Full lockup: HUZAIFA.DEV

### Core Value Proposition

Huzaifa leverages AI-augmented engineering to build applications that are **fast to ship**, **premium in quality**, and **built to scale** — without the traditional trade-off between speed and robustness.

| Pillar       | What It Means                                                                                                                               |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------- |
| **Fast**     | AI-accelerated scaffolding, refactoring, debugging, and code review compress timelines 2-3x. MVPs in weeks, not months.                     |

**The Result:** Clients get startup velocity with enterprise-grade quality. Faster time-to-market, lower defect rate, and architecture that doesn't need to be rewritten when traction hits.

## Content Types & Guidelines

### 1. Hero Section (`main_hero.md`)

**Purpose:** Instant positioning. Tells visitors what Huzaifa does and for whom in < 3 seconds.

**Structure:**

- `title`: A single bold declarative sentence (Oxanium, 84px vibe). Present tense. Positions as an architect/builder.
  - _Example:_ "Architecting modern web experiences with brutal precision."
- `description`: 1-2 sentences. Specific technologies and what they enable.
  - _Pattern:_ "I build [type of products]. Specializing in [tech 1], [tech 2], and [tech 3]."
  - _Optional AI angle:_ "Leveraging AI-augmented engineering for zero-defect delivery."
- `main_image`: Path to hero image.

### 2. Services (`services/*.md`)

**Purpose:** Capability showcase — what Huzaifa can deliver.

**Structure:**

- `number`: Two-digit string ("01"–"06")
- `icon`: Lucide icon name (code-2, zap, layers, palette, server, terminal)
- `title`: Short capability name. Format: "[Domain]". All-caps badge-adjacent.
  - _Examples:_ "Full-Stack Architecture", "SEO & Performance", "High-End Animation"
- `description`: One sentence. Captures both _what_ and _the outcome_.
  - _Pattern:_ "[Action] [systems] paired with [complementary capability]."
  - _Example:_ "High-performance backend systems paired with pixel-perfect frontend execution."
  - _Optional AI mention:_ "AI-augmented pipelines for rapid, bug-free delivery."
- `tags`: 3 all-caps tags representing tech or discipline.

**SEO Note:** Services are key landing pages for **local SEO** (e.g., "web developer Shahdara", "React developer Pakistan") and **service-based queries** (e.g., "full-stack development services", "SEO optimization for React sites").

### 3. Portfolio (`portfolio/*.md`)

**Purpose:** Proof of execution. Social proof + technical depth.

**Structure:**

- `title`: Project name (short, branded)
- `slug`: URL-safe version
- `order`: Display order (1-based)
- `url`: Live project URL (optional)
- `description`: 2-4 sentences. Covers: what the project does, who it's for, key technical highlights, and specific integrations/APIs used.
  - _SEO pattern:_ Include natural keywords: industry terms ("B2B vehicle provenance"), tech terms ("Next.js", "OpenAI API"), use case terms ("compliance platform", "LMS").
  - _Geo pattern:_ If project is location-specific (e.g., UK dealers), mention location naturally.
  - _AI mention (if relevant):_ "Delivered with AI-augmented engineering, ensuring rapid turnaround and minimal defect rate."
- `mainImage`, `images`: Paths to screenshots.
- `tools`: Array of tech used (as displayed chips).

**Portfolio Index (`-index.md`):**

- `title`: "ENGINEERED WORKS"
- `description`: A brutalist framing of the body of work.

### 4. Tech Stack (`section/tech-stack.md`)

**Purpose:** Trust signal for technical buyers. Shows breadth + depth.

**Structure:**

- `title`: "Tech Stack"
- `tag`: `// CORE_ARCHITECTURE`
- `description`: Curated positioning of the technology choices.
- `core_strengths`: 8 areas of engineering focus (Microservices, REST API, High-End Animations, Scalable System Design, Clean Code, Performance Optimization, Secure Integrations, Automation Workflows)
- `technologies`: Array of `{ name, icon }`. Currently 24 technologies across frontend, backend, infra, and tools.

**SEO Note:** This section ranks for **technology-specific queries** ("React developer", "Astro developer", "TypeScript expert"). Each tech name should be the canonical form used in search.

### 5. Testimonials (`testimonials/*.md`)

**Purpose:** Social proof from credible sources.

**Structure:**

- `quote`: A direct statement about working with Huzaifa. Focus on: problem-solving ability, technical depth, clarity, reliability.
  - _Pattern:_ Third party praising specific qualities (analytical thinking, engineering rigor, communication).
  - _Optional AI angle:_ "Delivered ahead of schedule with exceptional code quality."
- `author`: Full name.
- `role`: "Position, Company". Adds credibility.
- `order`: Sort order.
- `avatar_url`: Path to headshot.

**Geo Note:** If testimonial authors are local to Pakistan, it strengthens local SEO. If international, it strengthens authority for global queries.

### 6. Contact (`contact/-index.md`, `config/contact-form.toml`)

### 7. FAQ (`config/faq.toml`)

**Purpose:** Address objections and answer pre-sales questions.

**Structure:**

- Answers are direct, honest, and set clear expectations.

**SEO Note:** FAQ content naturally captures **long-tail voice search queries** and featured snippets. Each Q&A should use natural language matching how people actually ask.

## AEO (AI Engine Optimization) Guidelines

AEO = optimizing content so AI search engines (ChatGPT, Claude, Perplexity, Google AI Overviews, Bing Copilot) accurately parse, cite, and recommend the site.

### 1. Structured Data (JSON-LD) — Critical for AEO

AI engines heavily rely on structured data to understand entities and relationships.

**Required schemas:**

- **`Person`** — Huzaifa as the main entity. Properties: `name`, `jobTitle` ("Full-Stack Web Developer & Technical Architect"), `url`, `sameAs[]` (all social links), `knowsAbout` (array of technologies/skills), `address` (PostalAddress), `alumniOf` (if applicable), `award` (if any).
- **`WebSite`** — `name` "Huzaifa.Dev", `url`, `description`, `potentialAction` with SearchAction.
- **`WebPage`** — Per page, with `description`, `about`, `keywords`.
- **`LocalBusiness`** — Since there's a physical service area. `@type: "LocalBusiness"` with `areaServed: "PK"` and `address`.
- **`ItemList`** — For portfolio items, making each project a `ListItem` with `position` and `item` as a `CreativeWork`.

### 2. FAQ Schema

Wrap the FAQ section in `FAQPage` schema (Question + Answer pairs). Google AI Overviews frequently pulls from FAQ schema.

### 3. Conversational Content Patterns

AI search engines favor content that answers questions directly and conversationally.

**Patterns:**

- **Q&A format:** Use the FAQ section. Each answer should be a complete, self-contained answer (2-4 sentences).
- **"Who/What/Why/How" openings:** In descriptions, answer implicit questions.
  - _Example:_ "I build high-performance, accessible, and scalable digital products." → Answers "What does Huzaifa build?"
  - _Example:_ "Specializing in React, TypeScript, and modern CSS architecture." → Answers "What technologies does Huzaifa use?"
- **Definitions:** Clarify what services mean early in the description.
  - _Example:_ "Full-Stack Architecture: High-performance backend systems paired with pixel-perfect frontend execution."

### 4. Entity Recognition Optimization

AI models extract entities. Make them explicit:

- **People:** "Huzaifa" (main entity), client names (testimonials)
- **Technologies:** List explicitly in tech stack (24 technologies currently)
- **Locations:** Shahdara, Pakistan (explicit in address)
- **Services:** Full-Stack Architecture, High-End Animation, SEO & Performance, UI/UX Engineering, Scalable Systems, Digital Precision
- **Concepts:** "brutalist design", "GSAP animation", "technical architecture", "system design", "AI-augmented engineering", "prompt engineering"

### 5. Authoritativeness Signals for AI

AI engines assess E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness):

- **Experience:** Portfolio with real client projects, specific details (integrations, APIs, tools used)
- **Expertise:** Tech stack breadth (24 technologies), service variety, technical language precision
- **Authority:** Testimonials from named individuals with titles and companies, client logos/brands
- **Trust:** Contact page with full details, transparent FAQ, SSL/HTTPS

### 6. Content Formatting for AI Parsing

- Use clear heading hierarchy (H1 → H2 → H3)
- Keep paragraphs short (2-4 sentences)
- Use bullet points for lists (tech stack, tools, core strengths)
- Use bold for key terms (helps AI identify importance)
- Keep important information early in sections (inverted pyramid)

### 7. Avoiding AI Penalization

- **Do not** use excessive AI-generated fluff — AI detectors flag it and AI engines de-rank it
- **Do not** keyword stuff — use keywords naturally in context
- **Do not** use generic testimonials — they must be specific and attributable
- **Do** include specific numbers: "5 years experience, 42 projects, 28 clients"
- **Do** include specific tech: not just "modern stack" but "React, TypeScript, Node.js, GraphQL, Docker, AWS"
- **Do** include specific outcomes: not just "improved performance" but "reduced LCP from 4.2s to 1.1s"


_Example:_ "High-performance backend systems paired with pixel-perfect frontend execution."

```
A [type of application] for [target audience / industry].
The application [key function 1], [key function 2], and [key function 3].
Key features include [specific feature 1], [specific feature 2], [specific feature 3].
```
